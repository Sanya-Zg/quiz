Answers for the accessability quiz

üíª What does accessability stand for?

Accessibility refers to designing websites and applications so that everyone, including people with disabilities, can use them effectively.

üè∑Ô∏è What does 'WCAG' stand for?

WCAG stands for Web Content Accessibility Guidelines. These guidelines are a set of recommendations to make web content more accessible, especially for people with disabilities. They focus on making web content perceivable, operable, understandable, and robust for all users.

Example (no code needed): WCAG provides guidelines, such as ensuring color contrast and making content navigable with the keyboard.

üè∑Ô∏è Which element is used to provide alternative text for images for screen reader users?

The <img> element is used to embed images in web pages. The alt attribute within the <img> tag provides alternative text for screen readers. This allows users with visual impairments to understand the content of images by reading the description of the image instead of viewing it.

                    <img src="logo.png" alt="Company Logo">

Here, if a user is using a screen reader, the alternative text "Company Logo" will be read aloud instead of the image.

üè∑Ô∏è What does ARIA stand for in web development?

ARIA stands for Accessible Rich Internet Applications. It is a set of attributes that can be added to HTML elements to improve the accessibility of dynamic content and complex user interface controls. ARIA helps assistive technologies understand the role, state, and properties of user interface elements.

                    <button aria-label="Close" onclick="closeWindow()">X</button>

In this example, the aria-label attribute provides a description of the button's function, helping screen reader users understand its purpose.

üè∑Ô∏è Which of the following is not a principle of the WCAG?

The four key principles of WCAG are:

  • Perceivable: Users must be able to perceive the information presented.
  • Operable: Users must be able to interact with the content.
  • Understandable: Content must be clear and understandable.
  • Robust: Content must be compatible with current and future technologies.

Example (no code needed): WCAG defines these principles for making content more accessible.

üè∑Ô∏è Which of these color contrast ratios defines the minimum WCAG 2.1 Level AA requirement for normal text?

The WCAG 2.1 Level AA requirement for normal text is a contrast ratio of at least 4.5:1 between text and its background. This ensures that users with visual impairments, such as color blindness, can read the text clearly.

                    body {
                        color: #333333;
                        background-color: #ffffff;
                    }

The contrast ratio between dark gray text (#333333) and a white background (#ffffff) must meet the 4.5:1 requirement.

üè∑Ô∏è Which of the following elements is inherently focusable, meaning it can receive focus without a 'tabindex' attribute?

Elements that are inherently focusable include interactive HTML elements such as:

  • <a> (anchor tags with an href attribute)
  • <button>
  • <input>
  • <textarea>
                    <a href="https://www.example.com">Visit Example</a>

The <a> tag is inherently focusable and can receive focus when navigating through a page using the keyboard.

üè∑Ô∏è What is the purpose of the 'lang' attribute in an HTML page?

The lang attribute in HTML specifies the language of the content on the page. It helps assistive technologies, such as screen readers, correctly interpret and read the content aloud in the appropriate language.

                    <html lang="en">
                        <head>
                            <title>Accessible Page</title>
                        </head>
                        <body>
                            <p>Welcome to the accessible web page!</p>
                        </body>
                    </html>

Here, the lang="en" attribute specifies that the page's content is in English, helping screen readers use the correct pronunciation.

üè∑Ô∏è Which guideline ensures that content is accessible by keyboard as well as by mouse?

The operable principle of WCAG ensures that all content and functionality are accessible via keyboard, not just through mouse interaction. This principle ensures that users who cannot use a mouse can still navigate and interact with the content using only the keyboard.

                    <button onclick="alert('Button clicked!')">Click Me</button>

This button can be operated with both the mouse (by clicking) and the keyboard (by pressing the 'Enter' or 'Space' key).

üè∑Ô∏è What is the role of 'skip navigation' links in web accessibility?

Skip navigation links provide a way for users, especially those using screen readers or keyboard navigation, to skip repetitive navigation menus and jump directly to the main content of the page. This is particularly helpful for improving the efficiency of navigation for users with disabilities.

                    <a href="#maincontent" class="skip-link">Skip to main content</a>

In this example, the "Skip to main content" link allows keyboard and screen reader users to bypass the navigation and go straight to the primary content.

üè∑Ô∏è Which of these tools can help in checking the accessibility of a website?

There are several tools available for checking the accessibility of a website:

  • WAVE (Web Accessibility Evaluation Tool)
  • Lighthouse
  • Axe

These tools can help identify potential barriers and provide suggestions for improving accessibility to ensure websites are usable by all.
